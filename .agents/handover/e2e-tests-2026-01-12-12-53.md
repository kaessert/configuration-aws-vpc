#### Test 9: e2etest-vpc-ipv6-dual-stack - FAILED

**What happened:**
All 21 AWS managed resources created successfully, but VPC claim stuck at SYNCED=False, READY=False.

**Error:**
```
ReconcileError: cannot compose resources: pipeline step "vpc-resources" returned a fatal result:
failed to run kcl function pipelines: EvaluationError
  --> /src/subnets.k:90:1
   |
90 |     segment3Value = int(segment3Str, 16) if segment3Str and segment3Str != "" else 0
   |  called `Result::unwrap()` on an `Err` value: ParseIntError { kind: InvalidDigit }
```

**Root cause:**
IPv6 CIDR parsing bug in `functions/vpc/subnets.k:90` - the code attempts to parse IPv6 segment as hexadecimal without validating that all characters are valid hex digits (0-9, a-f, A-F).

**Fix required:**
```python
# Add hex validation before parsing
isValidHex = all(c in "0123456789abcdefABCDEF" for c in segment3Str)
segment3Value = int(segment3Str, 16) if isValidHex else 0
```
