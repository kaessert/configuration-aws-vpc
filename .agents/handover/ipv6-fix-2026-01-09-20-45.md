# Handover Report: IPv6 Dual-Stack VPC Late-Binding Fix

**Date:** 2026-01-09 20:45
**Project:** configuration-aws-vpc (Upbound Crossplane configuration)

## Current Problem

E2E test `e2etest-vpc-ipv6-dual-stack` was stuck for 38+ minutes with subnet creation failures. The root cause: subnets attempted to set `assignIpv6AddressOnCreation=true` before AWS assigned IPv6 CIDR blocks to the subnets, causing AWS API errors:

```
Cannot set assign-ipv6-address-on-creation to true unless the subnet has an IPv6 CIDR block associated with it
```

**Fix Implemented:** Late-binding pattern - extract VPC's observed IPv6 CIDR block and calculate subnet IPv6 CIDRs before setting the flag.

## What We Tried (That Didn't Work)

**Attempt 1: Two-phase composition tests with observed resources**
- Created phase 2 tests using `spec.observedResources` to mock VPC with IPv6 CIDR in status
- Expected: Composition would see observed VPC and calculate subnet IPv6 CIDRs
- Actual result: Phase 2 tests failed - composition didn't pick up observed resources
- Evidence: Subnets still generated without `ipv6CidrBlock` and `assignIpv6AddressOnCreation` fields
- Root cause: Composition test framework's `spec.observedResources` doesn't properly populate `ocds` for late-binding scenarios

**Attempt 2: Multiple status field access patterns in KCL**
- Tried `.get()` method: Not supported in KCL
- Tried multiline conditionals: KCL syntax errors
- Final working approach: Sequential boolean checks with single-line conditional

## Solution Implemented

### Code Changes

**1. `functions/vpc/main.k` (lines 293-306):**
```kcl
_vpcResources = [r for name, r in ocds if name == "vpc"] if ocds else []
_hasVpc = len(_vpcResources) > 0
_hasStatus = _hasVpc and "status" in _vpcResources[0] and _vpcResources[0]["status"]
_hasAtProvider = _hasStatus and "atProvider" in _vpcResources[0]["status"] and _vpcResources[0]["status"]["atProvider"]
_hasIpv6Cidr = _hasAtProvider and "ipv6CidrBlock" in _vpcResources[0]["status"]["atProvider"] and _vpcResources[0]["status"]["atProvider"]["ipv6CidrBlock"]

observedVpcIpv6CidrBlock = _vpcResources[0]["status"]["atProvider"]["ipv6CidrBlock"] if _hasIpv6Cidr else ""
```

Extracts VPC's observed IPv6 CIDR from `ocds` (observed composed resources). On first pass: empty string. On subsequent passes: AWS-assigned IPv6 CIDR.

**2. `functions/vpc/subnets.k` (lines 18-125):**

Added `_calculateSubnetIpv6Cidr()` function:
- Implements `cidrsubnet(vpcIpv6Cidr, 8, prefix)` logic in KCL
- Calculates /64 subnet blocks from VPC's /56 IPv6 CIDR
- Example: VPC `2600:1f13:d6f:5800::/56` + prefix 0 → `2600:1f13:d6f:5800::/64`

Updated `_buildIpv6Config()` to conditionally set IPv6 fields:
```kcl
if config.observedVpcIpv6CidrBlock and len(prefixList) > i:
    subnetPrefix = prefixList[i]
    subnetIpv6Cidr = _calculateSubnetIpv6Cidr(config.observedVpcIpv6CidrBlock, subnetPrefix)
    ipv6Config = ipv6Config | {ipv6CidrBlock = subnetIpv6Cidr}

    # Only set assignIpv6AddressOnCreation AFTER the IPv6 CIDR block is set
    if config.assignIpv6AddressOnCreation:
        ipv6Config = ipv6Config | {assignIpv6AddressOnCreation = True}
```

- AWS VPC IPv6: /56 CIDR block
- AWS Subnet IPv6: /64 CIDR blocks (8-bit increment)
