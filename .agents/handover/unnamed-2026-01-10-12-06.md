### Test Structure Pattern
Both IPAM tests follow the same pattern:
- Phase 1 (composition test): Validates VPC creation with IPAM config
- Phase 2 (E2E test): Validates late-binding with observed CIDR
- Subnets use prefix offsets instead of explicit CIDRs for late-binding

### IPAM IPv6 Late-Binding Flow
```
1. User creates VPC claim with:
   - ipv6IpamPoolId: "ipam-pool-ipv6-12345678"
   - ipv6NetmaskLength: 56
   - publicSubnetIpv6Prefixes: [0, 1]

2. First composition pass:
   - VPC created with IPAM config
   - No subnets created yet (no observed IPv6 CIDR)

3. AWS allocates IPv6 CIDR from IPAM pool:
   - Example: 2600:1f13:d6f:5800::/56
   - Appears in VPC status.atProvider.ipv6CidrBlock

4. Second composition pass:
   - Composition observes IPv6 CIDR
   - Calculates subnet CIDRs:
     - Prefix [0] → 2600:1f13:d6f:5800::/64
     - Prefix [1] → 2600:1f13:d6f:5801::/64
   - Creates subnets with calculated CIDRs
```
