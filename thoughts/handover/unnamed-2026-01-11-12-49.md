# Handover Report: Test Refactoring Plan

**Date:** 2026-01-11 12:49
**Project:** configuration-aws-vpc

## Current Problem

The test suite has massive code duplication (~85%) that needs refactoring to improve maintainability and development velocity. Currently:

- **88 test directories** (70 composition tests, 18 E2E tests)
- **174+ repeated providerConfigRef blocks** (identical structure)
- **88 repeated base spec configurations** (compositionPath, xrdPath, timeoutSeconds)
- **88 repeated import blocks** (6-8 imports per file)
- **18 repeated E2E ProviderConfig definitions** (identical web identity config)
- **30+ tests that could be consolidated** into ~10 test bundles

This creates high maintenance burden (update 88 times vs once) and slow test authoring (15-20 minutes vs 2-3 minutes).

## Context

Building an Upbound control plane configuration for AWS VPC (parity with terraform-aws-modules/vpc). The project uses Crossplane v2 with KCL composition functions. Tests are written in KCL using `up test` framework.

## Next Steps

Execute the refactoring plan starting with **P1: Create shared helper utilities**.

**P1 Details:**
- Create `tests/shared/helpers.k` with resource builder functions:
  - `buildManagedResource()` - Standard resource with providerConfigRef + managementPolicies
  - `buildCompositionTestSpec()` - Base spec with compositionPath, xrdPath, timeoutSeconds
  - `buildVPC()`, `buildSubnet()`, `buildNATGateway()`, etc. - Resource-specific builders
- Create `tests/shared/kcl.mod`
- Verify helpers work by updating 2-3 sample tests
- **MANDATORY:** Run `verify-configuration` skill to verify tests pass after changes

After P1 complete, continue with P2 (shared fixtures), then P3-P5 (consolidate test bundles).
